## 🎓 Learning Paths

### Path 1: "I want to use it today" (30 min)
1. [QUICK_START.md](QUICK_START.md) — 5 min setup
2. Run `npm run test:all` — 2 min
3. Open http://localhost:3000 — Explore pages
4. Done! You're using the MES

### Path 2: "I want to develop it" (2 hours)
1. [QUICK_START.md](QUICK_START.md) — Setup
2. [DEV_GUIDE.md](DEV_GUIDE.md) — Learn architecture
3. Try: Add a new API endpoint (15 min)
4. Try: Add a new UI page (20 min)
5. Run full test suite
6. Ready to contribute

### Path 3: "I want to deploy it" (4 hours)
1. [PRODUCTION_READINESS.md](PRODUCTION_READINESS.md) — Feature matrix
2. [DB_SETUP_GUIDE.md](DB_SETUP_GUIDE.md) — Postgres setup
3. [QUICK_START.md](QUICK_START.md) — Docker/cloud deployment
4. [HARDWARE_INTEGRATION.md](HARDWARE_INTEGRATION.md) — Hardware setup
5. Plan production rollout

### Path 4: "I need to integrate hardware" (Full Day)
1. [HARDWARE_INTEGRATION.md](HARDWARE_INTEGRATION.md) — Read edge pattern
2. [DEV_GUIDE.md](DEV_GUIDE.md) — MQTT bridge section
3. [QUICK_START.md](QUICK_START.md) — Test simulator
4. Design your edge gateway
5. Build real connectors following stubs in `lib/connectors/`
